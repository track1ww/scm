**보고일자**: 2026년 3월 27일

## 1. 개요

SCM 통합관리 시스템은 Phase 1에서 기본 PM(재무·회계) 모듈을 중심으로 시작했으나, 제조업 시장 진입을 위해 Phase 2에서 **SD(판매출하), HR(인사관리), PP(생산계획), QM(품질관리), TM(운송관리), WI(작업지시서), FI(세금계산서)** 등의 핵심 모듈을 대규모로 확장하는 중입니다.

현재 4개의 개발 트랙이 병렬로 진행 중이며, 각 트랙의 완료 목표는 **2026년 Q2 말**입니다.

## 5. Track별 개선 현황 상세 분석

### Track 1: SD(판매출하) & HR(인사관리) - 95% 완성

#### 5.1 SD 모듈 개선 현황

**목표**: 고객·수주·출하 전체 프로세스 자동화

**모델 구현 완료**:
- `Customer`: 고객 정보 (신용한도, 결제조건)
- `SalesOrder`: 판매주문 (수량, 단가, 할인)
- `Delivery`: 출하 (배송추적)

**API ViewSet 완료**:
```
GET/POST   /api/customers/              (고객 조회·생성)
GET        /api/customers/{id}/         (고객 상세)
PATCH      /api/customers/{id}/         (고객 수정)
DELETE     /api/customers/{id}/         (고객 삭제)

GET/POST   /api/sales-orders/           (수주 조회·생성)
GET        /api/sales-orders/{id}/      (수주 상세)
PATCH      /api/sales-orders/{id}/      (수주 수정)

GET/POST   /api/deliveries/             (출하 조회·생성)
```

**Serializer 완료**:
- CustomerSerializer
- SalesOrderSerializer (total_amount 계산식 포함)
- DeliverySerializer (order 관계 포함)

**프론트엔드 현황**:
- 페이지 구조: SdPage.jsx 완성 (탭 기반)
- 상태: "준비 중" 플레이스홀더
- **필요 작업**: 실제 API 연동 (조회, 생성, 수정, 삭제 폼)

**남은 작업** (2026년 4월 완료 목표):
1. SdPage.jsx에서 실제 API 연동 구현
2. 고객 관리 탭: 고객 목록, 신규 추가 폼
3. 판매주문 탭: 주문 목록, 할인 및 총액 자동 계산
4. 출하 처리 탭: 배송 상태 추적, 운송사 연동
5. KPI 카드 데이터 실시간 업데이트

### Track 2: PP(생산계획) & QM(품질관리) - 30% 예정

#### 5.3 PP(생산계획) 모듈 개선 계획

**목표**: BOM → 생산주문 → MRP 최적화

**계획 모델** (2026년 5월 말 완성 예정):
```
BOM (Bill of Materials)
  - part_code: 제품코드
  - component_code: 부품코드
  - quantity: 필요수량
  - unit_cost: 단위비용

ProductionOrder
  - order_number: 주문번호
  - product_code: 제품코드
  - quantity: 생산수량
  - start_date: 시작일
  - end_date: 완료예정일
  - status: (계획, 진행중, 완료)

MRPPlan (물자소요계획)
  - item_code: 품목코드
  - period: 계획기간
  - demand: 수요량
  - supply: 공급량
  - safety_stock: 안전재고
  - order_qty: 발주수량
```

**계획 API**:
- BOMViewSet: BOM 구조 관리
- ProductionOrderViewSet: 생산계획 입력·조회
- MRPPlanViewSet: MRP 자동 계산·조회

**특징**:
- MRP 자동 계산: 수요 → 필요자재 자동 식별
- 안전재고 관리
- 발주 자동 제안

**남은 작업**:
1. 모델 설계 및 마이그레이션
2. API ViewSet 구현
3. Serializer 작성
4. MRP 계산 로직 구현 (알고리즘)

---

#### 5.4 QM(품질관리) 모듈 개선 계획

**목표**: 입고검사 → 부적합 관리 → 개선조치

**계획 모델** (2026년 5월 말 완성 예정):
```
InspectionPlan
  - item_code: 검사대상 품목
  - inspection_type: (입고검사, 공정검사, 최종검사)
  - sample_size: 표본크기
  - accept_level: 합격수준
  - reject_level: 부적합수준

InspectionResult
  - plan: 검사계획
  - lot_number: 로트번호
  - inspected_qty: 검사수량
  - accepted_qty: 합격수량
  - rejected_qty: 부적합수량
  - result: (합격, 부적합)

DefectReport
  - inspection_result: 검사결과
  - defect_type: 부적합 유형 (치수, 외관, 기능)
  - defect_qty: 부적합 수량
  - action: 조치내용 (폐기, 재작업, 특별승인)
```

**계획 API**:
- InspectionPlanViewSet
- InspectionResultViewSet
- DefectReportViewSet

**특징**:
- 통계기반 샘플링 (AQL 기준)
- 부적합 원인 추적
- SPC(통계적 공정관리) 기초 구현

**남은 작업**:
1. 모델 설계
2. API 구현
3. 통계 계산 로직 (샘플링, OC 곡선)

#### 5.6 FI(세금계산서) 모듈 개선 계획

**목표**: 매출·매입 세금계산서 자동 생성 및 부가세 관리

**현재 상태**: Account, AccountMove 기본 구조 완성

**새로운 모델 추가**:
```
TaxInvoice (세금계산서)
  - issue_date: 발급일
  - issue_type: (매출, 매입)
  - supplier/customer: 거래처
  - total_amount: 공급가액
  - tax_amount: 부가세 (10% 자동 계산)
  - total_with_tax: 합계
  - status: (발급, 확정, 취소)
  - attachment_url: 파일 첨부

TaxDeduction (세액공제)
  - tax_invoice: 세금계산서
  - deductible_amount: 공제 가능 금액
  - deduction_date: 공제일

VATReturn (부가세 신고)
  - period_year, period_month
  - sales_tax: 매출세액
  - purchase_tax: 매입세액
  - payable_tax: 납부세액
  - status: (작성, 제출, 확정)
```

**계획 API**:
- TaxInvoiceViewSet (자동 생성, 조회, 확정)
- TaxDeductionViewSet
- VATReturnViewSet (부가세 신고 자동 계산)

**자동화 기능**:
- SD의 SalesOrder → TaxInvoice 자동 생성
- MM의 PurchaseOrder → TaxInvoice 자동 생성
- 월별 부가세 신고 자동 계산 (매출세액 - 매입세액 - 선급금)

**남은 작업**:
