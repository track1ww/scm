```
REST Convention:
  GET    /api/v1/modules/{resource}/           (List)
  POST   /api/v1/modules/{resource}/           (Create)
  GET    /api/v1/modules/{resource}/{id}/      (Retrieve)
  PUT    /api/v1/modules/{resource}/{id}/      (Update)
  DELETE /api/v1/modules/{resource}/{id}/      (Delete)

Response Format:
{
  "status": "success|error",
  "code": 200,
  "data": {...},
  "errors": [...],
  "timestamp": "2026-03-27T10:30:00Z"
}

Pagination:
{
  "count": 100,
  "next": "http://api/.../page=2",
  "previous": null,
  "results": [...]
}
```

- 인덱싱: 자주 조회되는 필드 (company, status, date)
- 파티셔닝: 거래 데이터는 월별 파티션 (향후)

### 보안 표준
```
Authentication:
  - JWT + Refresh Token (24h/7d)
  - 2FA (Google Authenticator)
  - Rate Limiting: 100 req/min

Authorization:
  - Role-Based Access Control (RBAC)
  - Module-level + Record-level 권한

Data Protection:
  - HTTPS/TLS 1.3 필수
  - 암호화: 계좌번호, SSN (AES-256)
  - 감시: 중요 거래 로그 (15년 보관)
```
